Right tool for the right job

Most common workflow for Transcriptions

Transcriptions for paper edit are more widely used in the industry then one might think, from realities like “wife swap” to factual entrainment documentaries for discovery or Nat Geo,  the workflow I’ve see most often involve a media player of some sort (usually quicktime) and the use of tabs in word.

and it consists of annotating the time code in the first column every 30seconds to 1 minute at most.

Smart Transcriptions: F5 Transcribe

However there is a software who can handle the time codes, and stamp them every time you hit enter to change line.
It’s called F5 Transcribe. It creates a rtf file with your transcriptions. There’s also the possibility to set shortcuts for stamping time-codes as well as speed, pause, and play to speed up the process. and if you over the cursor on the time code it lets you click on it and will jump to that part of the footage.



Personally I change line to stamp the time code every time the sentence spoken feels concluded, or there is a pause that would work for a cut, this makes it easier on making more accurate paper edits. However some just chose to make them every 30 seconds.
This is how I set the preferences for shortcuts, so that play/pause, and spool backward, and speed increase/decrease can all be done with the right hand. Decreasing the payback speed helps the level of accuracy, and overall transcription speed.

[tip: it can be download free of charge, it just requires and e-mail to sign up, it’s produced by German university professors who have built it as an aid for their research]

Formatting Using Word

The only downside of the time code generated by F5 is formatted like this:


Not only it has an # at beginning and end, but also instead of 00:00:23:00 it’s 00:00:23-0. So as you can see it has a dash - instead of :0 .
Reason to change this formatting is to make it faster when we are going to use the time-code as a reference in the paper edit to assemble the sequence in final cut pro (or equivalent editing system).


And here is how we go about doing it.
 

1. First of all we open the rtf transcription file with word

2.Open the “find and replace”  [edit > replace]

We find and replace - with :0 
[tip: when transcribing avoid using - during your transcription in the text, otherwise it would replace it there too]



3. then to eliminate the #. 

First we search # with a space to identify the second one and replace it with ^p

What next?

Now you can read your transcripts as they are, as a word document, print them etc.. or you can use word to format them even further to inline the column with the time codes, separated from a column with the speakers name and one with the transcribed speech, this can be done with very few clicks.  

However I am not going to dwell on that, because the workflow I prefer is that of using a Google Spreadsheet to craft the paper edit. an Even Smarter Transcriptions tool

ofcourse that is the solution I use, and there are other alternatives, for instance Adobe Premiere Pro has a speech recognition system that would  transcribe the footage for you. And each word is synced with the footage. obviously it might need proofing for accuracy, etc.. but depending on the project there might not be a need for very accurate transcriptions. It is also supposed to support multi language options which is always good.

Another option is using wistia or something equivalent to handle the transcriptions. it charges 5$ per minute, and uses a mixture of speech recognition and human transcribers, claiming a 100% level of accuracy.
